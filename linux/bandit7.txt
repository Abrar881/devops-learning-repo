## Bandit Level 6 → Level 7

**Challenge:** Find a file with these properties:
- Owned by user bandit7
- Owned by group bandit6
- 33 bytes in size

**Solution:**
```bash
find / -user bandit7 -group bandit6 -size 33c 2>/dev/null
cat /var/lib/dpkg/info/bandit7.password

**Explanation**
find / searches from root directory (/) instead of current directory (.), because the file could be anywhere on the server, not just in the home folder

-user bandit7 filters for files owned by user bandit7

-group bandit6 filters for files owned by group bandit6

-size 33c finds files exactly 33 bytes in size (the c means bytes)

2>/dev/null hides all the "Permission denied" error messages so only actual matches are shown

**Password:** [morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj]

**What I learnt** Searching the entire server means starting from / (root), not . (current directory)

Without 2>/dev/null, the output is cluttered with permission errors that make it hard to find the actual result

Usernames like bandit7 are one word, putting a space between bandit and 7 causes an error
