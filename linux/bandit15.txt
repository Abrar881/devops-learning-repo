## Bandit Level 15 → Level 16

**Challenge:** Submit the password for the current level (bandit15) to port 30001 on localhost using SSL/TLS encryption.

**Solution:**

Retrieve the current level password:
```bash
cat /etc/bandit_pass/bandit15
Output: 8xCjnmgoKbGLhHFAZlGE5Tmu4M2tKJQo

Connect to port 30001 using openssl with SSL/TLS:

bash
openssl s_client -connect localhost:30001
Once connected (after the certificate output and a blank line), paste the password:


8xCjnmgoKbGLhHFAZlGE5Tmu4M2tKJQo
The server responds with the next password:


 kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx

What I learned:

openssl s_client is used for SSL/TLS encrypted connections

The -connect option specifies the host and port to connect to

SSL/TLS encrypts the data being sent, unlike plain nc or telnet

The certificate information displayed is normal and can be ignored for this task

After connecting, you type the password just like with plain netcat
